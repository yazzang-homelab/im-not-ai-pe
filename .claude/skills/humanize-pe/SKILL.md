# humanize-pe 스킬

## 목표

기술 보고서를 **AI 냄새 없이, 공정엔지니어 문체로** 변환한다.

## 실행 순서

### 1단계: MCP 도구 확인

```
mcp__cng-ui__doc_humanize(file_path="<경로>", mode="v20pe")
```

- 반환값 `applied`: 이미 적용된 기계 규칙 목록
- 반환값 `content`: 기계 규칙 적용 후 텍스트 (이 텍스트에 2~3단계 적용)
- 반환값 `guidance`: 적용해야 할 판단 규칙 전문

### 2단계: v2.0 판단 규칙 적용

`guidance`의 v2.0 섹션에 따라:

1. **서술어 완성 (S1)**: 명사 종결 → 합쇼체
   - "발생함" → "발생합니다"
   - "필수적" → "필수적입니다"
   - 테이블 셀·제목·코드는 제외

2. **A-15 추정→단언 (S2)**: "~일 것으로 추정됩니다" → "~입니다"

3. **A-16 대명사 (S1)**: "그" → 구체 명사

### 3단계: PE 규칙 적용

`guidance`의 PE 섹션에 따라:

1. **배경 지식 삭제** (PE-1): 엔지니어 상식 수준 설명 제거
2. **설명문 → 명령형 불릿** (PE-2): `▶` 불릿, "~한다" 어미
3. **가설 소절 → 요약표** (PE-3): H1~HN 개별 서술 → 비교표
4. **분량 목표** (PE-5): 원본 대비 50~60%

### 4단계: 자체검증

- S1 패턴 0건 확인
- S2 ≤ 2건 확인
- 수치·테이블·Decision Gate 누락 없음
- 등급 A 달성 시 완료

## 출력 형식

HTML 보고서라면:
- 기존 CSS/레이아웃 유지
- 섹션별 변환 후 전체 파일 반환
- 파일명 `<원본명>_pe.html`로 저장

텍스트라면:
- 변환 전/후 비교 블록 제공
- 적용 규칙 목록 명시

## 유의사항

- 수치·스펙·기준값은 절대 변경 금지
- 테이블은 삭제 금지 (PE 압축이라도)
- 변경률 50% 초과 시 경고
