# 공정엔지니어(PE) 스타일 규칙 — v1.0

im-not-ai v2.0 위에 추가되는 공정팀·연구팀 보고서 전용 규칙입니다.

---

## 적용 우선순위

1. **im-not-ai v2.0 규칙** 먼저 적용 (이모지·번역투·서술어·쉼표 등)
2. **PE 규칙** 추가 적용 (압축·명령형·구조 변환)

---

## PE-1: 배경 지식 압축

**대상**: 독자(엔지니어)가 이미 아는 내용을 설명하는 단락

**기준**:
- 교과서 수준의 원리 설명 (예: "롤 라미네이션은 롤러를 이용해...")
- 제품 구조 설명 중 도메인 상식 수준의 내용
- "~이기 때문에", "~에 의해" 등의 인과 설명이 1문단 이상 이어질 때

**처방**:
- 삭제 가능: 완전 삭제
- 핵심 정보 포함: 1줄 요약 + 수치만 유지
- 테이블/그림으로 대체 가능 시: 산문 삭제, 테이블 유지

**유지 예외** (절대 삭제 금지):
- 수치, 스펙, 기준값
- 아직 알려지지 않은 가설/실험 결과
- Decision Gate 조건

---

## PE-2: 설명문 → 명령형 불릿

**대상**: 합쇼체 설명 산문

**처방**:
```
Before: "롤러 압력을 낮추어 단차부 에어갭을 저감합니다."
After:  ▶ 롤러 압력↓ → 단차부 에어갭 저감
```

**규칙**:
- `▶` 불릿으로 시작
- 서술어: "~합니다" → "~한다" 또는 명사 종결
- 이유 설명 절("~이기 때문에") 제거 — 결론만 유지
- 3줄 이상 연속 산문 → 불릿 리스트 또는 테이블로 전환

---

## PE-3: 가설 소절 → 요약표 통합

**대상**: 여러 가설(H1, H2, H3...)을 각각 소절로 서술한 구조

**처방**:
```
Before:
  4.2 H1 (유력) — 전구체 농도 변화
  [상세 설명 500자]
  4.3 H2 — 챔버 기류 불균일
  [상세 설명 400자]
  ...

After:
  | 가설 | 유력도 | 검증법 | 해결 레버 |
  |------|--------|--------|-----------|
  | H1   | 최유력 | 농도·점도 측정 | 습도 제어 복원 |
  | H2   | 유력   | 챔버 열화상 | 노즐 정렬 |
```

**보존 항목**:
- 최유력 가설의 핵심 메커니즘 1~2줄
- Decision Gate (가설 판별 기준)
- 해결 레버 우선순위

---

## PE-4: 서술어 명령화

**대상**: 행동 지시·절차 설명 문장

| Before | After |
|--------|-------|
| ~합니다 (행동) | ~한다 |
| ~을 확인합니다 | ~을 확인한다 |
| ~을 측정합니다 | ~을 측정한다 |
| ~을 도출합니다 | ~을 도출한다 |

**적용 범위**: 실험 계획, Phase 설명, 측정 항목, SOP

**예외**: 결론·기대효과 섹션은 합쇼체 유지 가능

---

## PE-5: 분량 목표

| 버전 | 목표 분량 | 용도 |
|------|----------|------|
| v2.0 윤문 | 원본의 90~95% | 언어 교정 |
| v2.0+PE | 원본의 50~60% | 팀 내 공유용 |
| PE 극압축 | 원본의 30~40% | 임원 보고용 |

**압축 우선순위** (먼저 제거):
1. 배경 설명 (PE-1)
2. 중복 이유 설명
3. 상식적 인과 문장
4. 장황한 가설 소절

**절대 압축 금지**:
- 수치·기준·스펙
- 테이블
- 리스크 대응
- 입력 대기/후속 조치 항목
